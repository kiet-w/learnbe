Phân Tích Chuyên Sâu Module Auth (src/auth)

Tài liệu này cung cấp một cái nhìn chuyên sâu vào cách thức hoạt động của module Xác thực (Auth). Trọng tâm là sự tương tác giữa Controller và Service, dòng chảy dữ liệu và vai trò của các thư viện bên thứ ba.

1. Controller (src/auth/auth.controller.ts)

AuthController là cửa ngõ giao tiếp, tiếp nhận yêu cầu HTTP và áp dụng các lớp bảo vệ thông qua Decorators.

1.1. Giải Mã Các Decorators Trọng Yếu

@UseInterceptors(ClassSerializerInterceptor)
Tự động hóa quá trình "làm sạch" dữ liệu nhạy cảm (như password) trước khi gửi về client.
@Res({ passthrough: true })
Cho phép thiết lập Header/Cookie mà không làm gián đoạn quy trình xử lý return của NestJS.
@UseGuards(AuthGuard)
Bảo vệ endpoint, trích xuất và verify JWT từ cookies để gắn vào request.user.

2. Service (src/auth/auth.service.ts)

AuthService chứa logic cốt lõi: băm mật khẩu, sinh token và thiết lập cookies.

2.1. Các Phương Thức Public

  • register: Kiểm tra email, băm mật khẩu qua bcrypt, lưu user mới.
  • login: So khớp password, sinh cặp Access/Refresh Token, thiết lập Cookies.
  • handleRefresh: Kiểm tra Refresh Token trong DB (chống Token Hijacking), cấp mới cặp token.
  • handleLogout: Vô hiệu hóa token trong DB và xóa Cookies trình duyệt.

2.2. Các Phương Thức Private

setCookies()
Cấu hình bảo mật Cookie: httpOnly, secure, sameSite.
generateTokens()
Sử dụng jsonwebtoken để ký mã với Secret Key và TTL tương ứng.

4. Vai Trò Cốt Lõi Của Các Thư Viện

  • bcryptjs: Mã hóa mật khẩu một chiều với salt rounds = 10 (cân bằng bảo mật/hiệu suất).
  • jsonwebtoken: Cơ chế Stateless Session với Access Token (15m) và Refresh Token (7d).
  • class-transformer: Lá chắn bảo mật cuối cùng giúp ngăn chặn rò rỉ dữ liệu nhạy cảm.
