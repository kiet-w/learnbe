Giải Thích Chi Tiết Chuyên Sâu - Module Catalog (src/catalog)

Module Catalog là "quầy trưng bày" của hệ thống e-commerce, chịu trách nhiệm xử lý các luồng dữ liệu liên quan đến danh mục, sản phẩm, tìm kiếm và lọc. Đặc thù của module này là lượng truy cập đọc (read-heavy) rất lớn, do đó kiến trúc của nó tập trung tối đa vào tối ưu hoá hiệu năng (Performance) và bộ nhớ đệm (Caching).

1.2 Cơ Chế Ánh Xạ URL Query Vào DTO (@Query() query: ProductQueryDto)

@Query()
Lấy tham số từ URL Query String và map vào object.
Tác động: Tự động instantiate ProductQueryDto thông qua ValidationPipe.

Trong endpoint GET /products, NestJS sử dụng Global ValidationPipe để tự động khởi tạo object. ProductQueryDto kế thừa từ PaginationQueryDto giúp tái sử dụng logic phân trang hệ thống.

2. Services: Lõi Logic Trọng Tâm (catalog.service.ts)

CatalogService thực thi Business Logic, tách biệt public API và private helpers.

2.1 Các Phương Thức Public

  • findCategories(): Trả về danh mục, tận dụng Redis getOrSet.
  • findProducts(query): Điều phối phân trang, tạo cache key, thực thi song song Prisma query qua Promise.all.
  • findProductBySlug(slug): Lấy chi tiết qua Cache, validate trạng thái ACTIVE.
typescriptcatalog.service.ts
async findProducts(query: ProductQueryDto) {
  const { page = 1, limit = 10 } = query;
  const skip = (page - 1) * limit;
  const cacheKey = this.getProductsCacheKey(query, page, limit);

  return this.redisService.getOrSet(cacheKey, async () => {
    const where = this.buildProductWhere(query);
    const [items, total] = await Promise.all([
      this.prisma.product.findMany({ where, skip, take: limit }),
      this.prisma.product.count({ where })
    ]);
    return { data: items.map(p => this.serializeProduct(p)), total, page, limit };
  });
}

2.2 Tương Tác Với Các Private Helpers

serializeProduct
Data Mapper chuẩn hóa dữ liệu Prisma sang JSON.
Quan trọng: Biến đổi Decimal sang string để bảo toàn độ chính xác tài chính.
  • buildProductWhere: Chuyển DTO sang Prisma.ProductWhereInput.
  • getProductOrderBy: Dịch enum sort sang Prisma order object.
  • getProductsCacheKey: Sinh unique key dựa trên query params.

3.2 Tác Động Chéo Giữa Các Module

Module Catalog phụ thuộc hoàn toàn vào Redis Cache. Trách nhiệm làm mới dữ liệu thuộc về Admin Module. Khi Admin thực hiện CUD, Admin Service sẽ invalidate cache keys tương ứng.

4. Tích Hợp Third-Party & Thư Viện

4.1 Tối Ưu Bằng Promise.all Với Prisma

Sử dụng Promise.all() để thực thi song song findMany và count, giảm gần một nửa thời gian chờ I/O.

4.2 Pattern Cache-Aside Với redisService.getOrSet

Cơ chế: Kiểm tra Redis (Hit) -> Trả ngay. Nếu Miss -> Query DB -> Lưu Redis (TTL) -> Trả dữ liệu. Đảm bảo hệ thống "đỡ" được traffic CCU lớn.
